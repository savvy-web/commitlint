# Commit Types

`@savvy-web/commitlint` extends the standard conventional commit types with
additional types for AI workflows and releases.

## Available Types

| Type | Description | Example |
| ---- | ----------- | ------- |
| `ai` | AI/LLM agent document updates | `ai: update CLAUDE.md with new context` |
| `build` | Build system or dependencies | `build: update webpack configuration` |
| `chore` | Maintenance tasks | `chore: update dependencies` |
| `ci` | CI/CD configuration | `ci: add GitHub Actions workflow` |
| `docs` | Documentation changes | `docs: update API reference` |
| `feat` | New features | `feat: add user authentication` |
| `fix` | Bug fixes | `fix: resolve memory leak in cache` |
| `perf` | Performance improvements | `perf: optimize database queries` |
| `refactor` | Code restructuring | `refactor: extract validation logic` |
| `release` | Release commits | `release: v1.2.0` |
| `revert` | Revert previous commits | `revert: undo feature X` |
| `style` | Code formatting | `style: fix indentation` |
| `test` | Test changes | `test: add unit tests for parser` |

## Custom Types

### ai

For AI/LLM agent document updates such as:

- CLAUDE.md context files
- AI agent configuration
- Prompt templates
- Context documentation

```text
ai: update development context with new patterns
ai(context): add API usage examples for agents
```

### release

For release commits generated by version management tools:

```text
release: v1.2.0
release: version packages
release(api): v2.0.0
```

## Commit Message Format

```text
<type>(<scope>): <subject>

<body>

<footer>
```

**Components:**

- **type** - One of the allowed types (required)
- **scope** - Affected area, usually a package name (optional)
- **subject** - Short description in imperative mood (required)
- **body** - Detailed explanation (optional)
- **footer** - References, breaking changes, signoff (optional)

## Examples

### Feature with scope

```text
feat(api): add rate limiting support

Implements token bucket algorithm for API rate limiting.
Configurable limits per user tier.

Closes #123
Signed-off-by: Developer Name <dev@example.com>
```

### Bug fix

```text
fix: prevent crash on empty input

Handle edge case where user provides empty string.

Signed-off-by: Developer Name <dev@example.com>
```

### AI context update

```text
ai: refresh development context after refactoring

Update CLAUDE.md to reflect new module structure
and coding patterns established in recent refactoring.

Signed-off-by: Developer Name <dev@example.com>
```

### Breaking change

```text
feat(api)!: change authentication method

BREAKING CHANGE: API now requires JWT tokens instead of
API keys. See migration guide for details.

Signed-off-by: Developer Name <dev@example.com>
```

## DCO Signoff

When DCO is enabled (default when a `DCO` file exists), all commits must
include a signoff:

```text
Signed-off-by: Your Name <your.email@example.com>
```

Add with `git commit -s` or configure your Git client to add automatically.

## Programmatic Access

Export commit types for use in your own tools:

```typescript
import {
  COMMIT_TYPES,
  COMMIT_TYPE_DEFINITIONS,
} from "@savvy-web/commitlint";

// Array of type strings
console.log(COMMIT_TYPES);
// ["ai", "build", "chore", "ci", "docs", "feat", ...]

// Detailed definitions with descriptions
console.log(COMMIT_TYPE_DEFINITIONS);
// [{ type: "ai", description: "...", title: "AI Context" }, ...]
```
